## What Are Handovers?

Handovers are snapshots of session context that prevent "amnesia" between sessions. Each handover captures:
- What was accomplished
- Key decisions made
- Problems encountered
- Lessons learned
- What's next

**When to Create a Handover:**
- Before ending a session with incomplete work
- When context window is getting full
- Before switching to a different feature
- After major milestones (even if complete)

**How to Create:** Run `/speckit.handover`

**How to Resume:** Run `/speckit.resume`

## Tips for Effective Handovers

1. **Be Specific**: Include file paths, line numbers, exact decisions
2. **Be Honest**: Record what went wrong, not just wins
3. **Be Actionable**: Next TODOs should be clear and prioritized
4. **Be Contextual**: Explain why decisions were made, not just what
5. **Link Everything**: Reference spec, plan, tasks, napkin entries

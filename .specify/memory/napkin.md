## How to Use This File

This napkin serves as a **living record** of implementation wisdom. Every mistake is a learning opportunity.

**When to Record:**
- Implementation bugs or errors
- Specification ambiguities that caused rework
- Planning oversights or architectural mistakes
- Test failures that revealed design flaws
- Constitution violations
- Performance issues or optimization opportunities

**Format:**
Each entry should include:
1. **Timestamp & Category** - When and what type of issue
2. **Context** - Feature/file/component affected
3. **What Went Wrong** - Clear description of the problem
4. **What Fixed It** - The solution or correction
5. **Lesson Learned** - Generalized principle to apply going forward
6. **References** - Links to relevant commits, handovers, or documentation

**Categories:** SPEC | PLAN | IMPLEMENTATION | TEST | CONSTITUTION | ARCHITECTURE | PERFORMANCE
